## Critical Issues

### 1. Stale deployment
See above. Deploy before anything else.

### 2. Pure client-side rendering (CSR)
Live HTML body is an empty `<div id="root">`. Googlebot eventually renders JS (delayed, second-wave indexing), but **GPTBot, ClaudeBot, and PerplexityBot do not execute JavaScript** — AI search engines see almost nothing.

Platform-specific AI readiness scores:

| Platform | Score | Reason |
|---|---|---|
| Google AI Overviews | 38/100 | Schema readable; CSR body invisible until delayed JS render |
| ChatGPT Search | 22/100 | No JS execution; sees only noscript + schema; no llms.txt live |
| Perplexity | 25/100 | Crawls raw HTML only |
| Bing Copilot | 30/100 | Limited JS rendering; schema/OG give partial signal |

- **Long-term fix:** prerender/SSG (e.g., `vite-ssg` or a build-time prerender pass)
- **Interim fix:** deploy + expand the `<noscript>` block to include FAQs, the three stats, B2B/B2C descriptions, and the six process steps (currently covers ~15% of page content)

### 3. www redirect is 307 (temporary), not 308/301
`https://www.mimicstudio.co` → 307 → `https://mimicstudio.co/`. Google does not treat 307 as a permanent canonical signal. vercel.json has `"permanent": true` but live serves 307 — verify Vercel dashboard domain config after redeploying. (http → https is correctly 308.)

### 4. 34 MB hero video with `preload="auto"`
`/frames/Grasslands0001-0320.mp4` is 34 MB — by far the heaviest asset. Compress to under ~3 MB (lower bitrate, shorter loop, or poster image + lazy load) and switch to `preload="metadata"`.

## High Priority

### 5. 2,170 ms artificial loading screen
`MIN_LOADER_MS = 2170` in `src/App.jsx` intentionally hides all content for 2.17s, pushing LCP past the 2.5s "good" threshold before the network even matters. Reduce to 0 or don't let it gate the LCP element.

### 6. 877 KB JS bundle (~277 KB compressed transfer)
Heavy for a single-page marketing site; must parse/execute before anything renders (LCP + INP impact). Code-split: lazy-load below-fold sections (Solutions, FAQ) and the animation library. Target < 200 KB initial chunk.

### 7. Broken asset caching
Hashed `/assets/*.js|css` files are served with `Cache-Control: public, max-age=0, must-revalidate`. Hashed assets should be `max-age=31536000, immutable`. Check whether a vercel.json `"/(.*)"` headers rule is clobbering Vercel's default asset caching.

### 8. No portfolio, testimonials, or team identity (E-E-A-T composite: 21.5/100)
- Zero work samples — most disqualifying trust gap for a web design agency
- Stats ("98% satisfaction", "3.2x conversions", "10–14 days") have no attribution or methodology — unverified claims are a QRG red flag
- No founder/team name, no stated location, no "who is responsible for this site"
- Total rendered copy ≈ 715 words — ~30% below the practical floor for a homepage doubling as the sole service page (target 1,200–1,800)

**Top content fixes:**
1. Portfolio grid with 3–5 project entries (industry, screenshot, 2–3 outcome sentences; client names optional)
2. 3+ attributed testimonials with measurable outcomes (place after the stats bar to substantiate the claims)
3. 100–150-word founder/about passage (name, background, where the studio operates)

## Medium Priority

### 9. Schema regression in local source
Local `index.html` changed `ProfessionalService` → `Organization`, losing local-business rich result eligibility. Fix:
- Restore `@type: "ProfessionalService"` (keep the `@id` graph improvements)
- Add `sameAs` (real social profiles only), `logo` (ImageObject), `image`
- Remove invalid `"AdministrativeArea": "Worldwide"` from `areaServed` (not a valid Schema.org place — list real countries instead)
- Optionally add `geo` and `openingHoursSpecification` if a physical anchor/hours exist
- **FAQPage:** keep it — no Google rich results for commercial sites (Aug 2023 restriction), but it has AI/LLM citation value. Info priority only.
- Add `aggregateRating` only once real, verifiable reviews exist

### 10. H2 headings are all slogans
No keyword-bearing noun phrases anywhere in the heading outline. Rewrite/supplement:
- "The difference is everything" → "Why Businesses Choose Mimic.Studio for Web Design"
- "Built for B2B and B2C" → "B2B and B2C Website Design Services"
- "Questions, answered" → "Web Design FAQ"

### 11. llms.txt format non-compliant
Missing the spec's `> blockquote` summary and `- [Title](URL)` link lists under H2 sections. Content is accurate and useful; fix format, then deploy.

### 12. No Content-Security-Policy header
Not in vercel.json at all. Add one covering script-src, style-src (Google Fonts), font-src (gstatic), connect-src, media-src, `frame-ancestors 'none'`.

### 13. No pricing transparency
Copy promises "a clear quote… no hidden fees" but gives no range. Even "projects typically range from X–Y" in the pricing FAQ answer removes the largest conversion friction and a QRG low-quality signal.

### 14. Sitemap — ✅ fixed locally during this audit
`public/sitemap.xml` was corrected: `lastmod` set to the real content date (2026-05-29), ignored `changefreq`/`priority` tags removed. Deploys with everything else.
- Keep privacy/terms **out** of the sitemap; add `<meta name="robots" content="noindex, follow">` to both pages
- Resubmit in Google Search Console after deploy

- robots.txt correct and fully open to all AI crawlers (GPTBot, ClaudeBot, PerplexityBot, CCBot, etc.)

- HTTPS everywhere; HSTS present (2-year max-age)
- Schema lives in raw HTML `<head>`, not JS-injected — readable even by non-JS crawlers
- Fast TTFB (~190 ms from Singapore edge); HTML payload only 3.2 KB
- Brand naming is consistent (`Mimic.Studio` in all machine-readable fields)
- The local codebase is in much better shape than the live site — deploying is the highest-ROI action available
